## 로그와 메트릭

**로그**: 서버가 동작할 때 서버의 상태와 동작 정보를 시간 경과에 따라 기록된 결과

**메트릭**: 시스템의 성능과 상태에 대한 통계적인 정보

---

**메트릭**

- 메트릭을 잘 수집하면 시스템의 현재 상태를 손쉽게 파악할 수 있고,  
  사업 현황에 관한 유용한 정보를 얻을 수 있음

- ex) DAU, Retension, CPU 사용량, 메모리 사용량 등

- 이렇게 메트릭을 수집하여 서비스가 현재 안정적으로 동작하고 있는지 파악하기 위해 사용할 수 있음

- 또한, 톰캣 스레드 풀과 커넥션 풀의 상태와 error 레벨 로그를 수집한 이유는  
  서버 프로그램 내부에 비정상적인 상황이 생기는 것을 신속히 대응하기 위함

쿼리 매트릭을 사용하여 DB에서 실행되는 SQL 들이 얼마나 자주, 오래, 무겁게 사용하는지 확인할 수 있음.

현재 사용중인 **ASH Viewer가 그 메트릭, 스냅샷이었음.**
