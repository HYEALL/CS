## 트랜잭션 격리 수준
트랜잭션 격리 수준에 따라

- **낮은 격리 수준** : 높은 처리 능력, 데이터 일관성 문제가 생김

- **높은 격리 수준** : 낮은 처리 능력, 데이터 정합성 굿


READ UNCOMMITTED : 커밋되지 않은 트랜잭션 다른 트랜잭션이 조회 가능 - Dirty / Non-repeatable / Phantom 문제 발생

READ COMMITTED : 완료된 것만 다른 트랜잭션이 조회 가능 - Dirty 방지, Non-repeatable / Phantom 문제 발생

REPEATABLE READ : 한 트랜잭션에서 특정 레코드 조회할 때 항상 같은 데이터 응답 보장하지만 행 추가는 가능 - Dirty / Non-repeatable 방지, Phantom 문제 발생

SERIALIZABLE : 모두 접근 불가 - Dirty / Non-repeatable / Phantom 방지


/ - dirty read 는 아직 commit 하지 않은 변경 중인 데이터를 읽는 경우

/ - phantom read 는 한 트랜잭션이 동일 쿼리를 두 번 실행했는데 결과 바뀌는 경우( insert, delete 시 발생 )

/ - non repeatabled read 는 같은 트랜잭션 안에서 동일한 쿼리를 실행했는데 다른 결과를 얻는 경우 ( update 발생 )


<u>**Oracle의 격리 수준**은 Read commit이므로 commit 완료된 것만 다른 트랜잭션이 조회할 수 있어 Dirty read는 발생하지 않지만 나머지는 발생해</u>
Oracle은 MVCC(Undo 기반)로 미커밋 데이터는 절대 안 보이므로 Dirty Read가 안 생김
